Question 1: Credit Card Checker

This question is worth 30% of the assignment mark.

Translate the program CreditCardChecker.java to C, including automated testing. You can use the code from ccc.c as a starting point, where the code for testing has already been translated.

Notes:

  • Even though the code can trivially be collapsed into a single function, use the same functions for the C implementation as in the Java code.
  • There is no Math.abs() in C. You can implement its behaviour in this program in any appropriate fashion. Simple is OK.
  • Use exactly the same test cases. Your program must pass exactly the same tests. The C output must be identical to the Java version.

Question 2: Longest Duplicate Subsequence

This question is worth 30% of the assignment mark.

This problem will be familiar to those of you who have taken COMP 1020 in Winter 2023. Translate the posted version of the model solution from that term LDS.java to C, including automated testing.

Notes:

  • Use exactly the same test cases. Your program must pass exactly the same tests. The C output must be identical to the Java version.

Question 3: Word Reverser

This question is worth 40% of the assignment mark.

Translate the Reverser.java Java program into a C program that performs identical processing, including automated testing.

Notes:

  • Re-write the Java functions as equivalent C functions. The parameters you pass to the C functions may change.
  • The Java functions are very "non C"-like. You can change the reversing function to either reverse the string in-place (modifying the incoming string), or be passed a destination string as a parameter.
  • There is no StringBuilder in C. You can replace the calls to .reverse() with your own loop, which you may want to place in a separate function.
  • IMPORTANT: Read the two points above carefully. Don't try to write your solution using the C equivalent of string concatenation or .substring()!
  • Assume a maximum string length of 256 characters, counting the \0
  • You can find an equivalent to Character.isLetterOrDigit() on the CS Linux system by entering man isalnum
  • The two variables used to count tests can be global variables. Don't add any other globals.
  • Use exactly the same test cases. Your program must pass exactly the same number of tests. Note that one test will fail in each of the Java and C programs!
