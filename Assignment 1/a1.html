Question 1

Convert the Duplicates.java Java program into a C program that performs identical processing, including automated testing.

Notes:

  • Re-write the Java methods as equivalent C functions. The parameters you pass to the C functions may change.
  • You can find an equivalent to Character.isWhitespace() on the CS Linux system by entering man isalpha
  • Use exactly the same test cases. Your program must pass exactly the same number of tests.
  • All the C code for this question must be in a single source file.

Question 2

Convert the recursive Solver.java Java program into a C program that performs identical processing (i.e. still using recursion).

Here is one sample data set that you can use to test the program.

2 2 1 3
5 1 2 3 4 5 9
1 2 3

Notes:

  • File processing must use the "safe" C strategies described in class (at least, as safe as the sample program).
  • Part of the problem is figuring out what the purpose of the code is, to come up with your own test data. Otherwise, treat it like a black box. The markers will test it on a larger data set, which you will not be given.
  • The output from your C program must be identical to the output of the Java program for any given input. Your C program does not need to handle any input errors that are not also handled by the Java program.
  • All the C code for this question must be in a single source file.
